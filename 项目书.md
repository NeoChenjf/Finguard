# FinGuard 项目书

## 1. 产品定位
FinGuard（C++）是一个面向个人资产配置的高性能 AI 网关与风控中台，提供流式投研问答、资产配置建议、组合风控闸门与再平衡计划，并具备限流/熔断/缓存/可观测/压测与性能优化能力。

## 2. 产品目标
- 为个人投资者提供稳定、可追溯的配置建议与风控输出。
- 以 C++ 中台承接高并发与可靠性治理，确保服务可用性与可控输出。
- 以可观测与压测数据驱动容量规划与性能优化。

## 3. 目标用户与场景
- 目标用户：有基础资产配置需求的个人投资者、理财顾问工具集成方。
- 核心场景：
  - 获取配置建议与再平衡计划。
  - 在问答过程中实时插入“风控提示”和“引用来源”。
  - 在波动环境下根据规则自动修正组合。

## 4. 核心架构（HTTP + 流式）
- 客户端：Web / 微信小程序 / curl
- C++ API 服务（核心）：HTTP + SSE 流式输出
- Python/脚本：拉行情数据、生成特征、更新数据库（可选）
- 存储：Postgres（或 SQLite 先跑通） + Redis（可选）

**架构职责划分**：
- C++ 服务为“中台”，负责并发、可靠性、风控与输出规范。
- Python/Java 仅用于数据处理与前端集成。

## 5. 核心模块
### 5.1 请求流水线（Pipeline）
请求 → 画像解析 → 数据检索 → 候选方案生成 → 风控闸门 → 报告流式输出

### 5.2 风控闸门（Risk Gate，C++ 必须实现）
- 单资产/单类占比上限
- 最大回撤阈值（用历史数据估算）
- 波动率目标范围
- 应急金比例（现金占比下限）
- 杠杆/高风险资产禁用开关（按风险偏好）

**标准输出**：
- PASS / WARN / FAIL
- 被触发的规则列表
- 自动修正后的组合（可选）

### 5.3 可靠性（限流/熔断/重试）
- 令牌桶限流：按用户 / 按接口 / 按模型
- 熔断器：外部模型/数据源失败率高则短路
- 超时 + 重试 + fallback（从高成本模型降级）

### 5.4 可观测与压测
- Metrics：请求数、P50/P95/P99、外部调用耗时、缓存命中率、熔断次数
- 压测：wrk / hey，记录吞吐与延迟数据

## 6. HTTP 接口设计
### 6.1 配置建议（非流式）
**POST** `/api/v1/plan`

输入示例：
```json
{
  "profile": {
    "risk_level": "medium",
    "horizon_months": 36,
    "cashflow_monthly": 3000,
    "emergency_fund_months": 6
  },
  "portfolio": [
    {"symbol":"SPY","weight":0.30},
    {"symbol":"BND","weight":0.40},
    {"symbol":"GLD","weight":0.10},
    {"symbol":"CASH","weight":0.20}
  ],
  "constraints": {
    "max_single_asset": 0.35,
    "max_drawdown": 0.20
  }
}
```

输出字段：
- `proposed_portfolio`（建议权重）
- `risk_report`（风险指标、触发规则）
- `rationale`（理由，含引用/数据来源）
- `rebalancing_actions`（可执行清单）

### 6.2 投研问答（流式 SSE）
**POST** `/api/v1/chat/stream`

`text/event-stream`，每个 event：
- `type`: token / cite / metric / warning / done
- `payload`: 内容

### 6.3 数据更新（可选）
**POST** `/api/v1/data/refresh`

触发 Python ETL 更新行情（可先 mock）。

## 9. 交付与上线要求
- 服务可通过 `docker compose up` 一键启动。
- README 必须包含：架构图、接口示例（curl）、压测结果（P95、吞吐）、风控规则示例（rules.yaml）、关键设计说明（线程模型、限流熔断、缓存策略）。
- 关键链路具备可观测性与可追溯日志（含 trace_id）。

## 10. 阶段路线
**第一阶段（服务骨架与基础接口）**：Drogon 起服务；`/health`、`/api/v1/plan` 返回 mock。

**第二阶段（流式问答与模型接入）**：接入 SSE：`/api/v1/chat/stream`；实现基础 LLM client。

**第三阶段（风控规则与组合校验）**：风控规则引擎（`rules.yaml`）；组合校验：`max_single_asset`、现金下限、风险偏好禁用清单。

**第四阶段（可靠性与可观测）**：缓存（LRU）+ 限流（令牌桶）；请求 `trace_id` + 结构化日志。

**第五阶段（压测与性能剖析）**：压测（wrk/hey）并记录；perf/heaptrack profiling。

**第六阶段（测试加固与 Release 构建）**：gtest 单测补齐关键路径；Release 模式编译；新增设置接口（`/api/v1/settings`）支持前端动态配置 API Key 与模型 URL；文档完善。

**第七阶段（前端 MVP — Web UI）**：Vite + React + TypeScript 前端；对接 `/plan` + `/chat/stream` 接口；SSE 流式渲染；设置页（API Key / 模型 URL 输入与持久化）；配置建议结果与风控报告展示。

**第八阶段（桌面壳与本地启动器 — Tauri）**：Tauri v2 打包为 Windows 桌面应用；finguard.exe 作为 sidecar 自动拉起；健康检查与服务状态显示；托盘图标与启动/停止控制；配置持久化至用户目录。

**第九阶段（打包分发与最终交付）**：Windows 安装包（MSI / NSIS）或便携版生成；内含 finguard.exe + 前端资源 + 默认配置；首次使用引导（填入 API Key → 验证连通 → 开始使用）；端到端验收（干净系统安装 → 配置 → 获取投资建议）；用户手册。

**第十阶段（Value Cell 量化价值分析系统）**：放弃 PDF RAG 路径，转型为"定性 + 定量"量化价值投资分析系统。通过 Yahoo Finance API 获取结构化财务指标（PE / PB / PEG / 预期增长率），通过搜索引擎 API 获取护城河、管理层与商业模式定性信息。实现安全边际计算与量化评价逻辑，输出 Markdown 结构化分析报告。

## 11. 阶段 6-10 详细规划

### 11.1 第六阶段：测试加固与 Release 构建

**目标**：确保后端稳定可交付，补齐前端对接所需接口。

**交付物**：
1. **gtest 单元测试**：覆盖风控规则引擎、LRU 缓存、LLM client 解析、pipeline 核心路径。
2. **Release 编译**：CMake Release profile，验证性能不退化。
3. **设置接口（新增）**：
   - `GET /api/v1/settings` — 返回当前可配置项（模型名称、URL，API Key 是否已配置）。
   - `POST /api/v1/settings` — 运行时更新 API Key、模型 URL、模型名称，写入 `config/llm.json` 并热生效。
4. **文档完善**：README 含架构图、接口示例（curl）、压测数据、风控规则示例。

**验收标准**：
- gtest 全部通过，关键路径覆盖率 > 80%。
- Release 构建可运行，`/health`、`/plan`、`/chat/stream`、`/settings` 均正常。
- 设置接口可成功更新 API Key 并立即在下次请求中生效。

**优先级说明**：Phase 6 执行"最小必要"原则——设置接口与 Release 构建优先，gtest 补齐关键路径即可，不追求 100% 覆盖。目标是尽快解锁 Phase 7 前端开发。

### 11.2 第七阶段：前端 MVP（Web UI）

**目标**：实现可用的 Web 前端界面，用户可通过浏览器与 FinGuard 交互。

**技术栈**：Vite + React + TypeScript + TailwindCSS（或类似 CSS 框架）。

**功能范围（MVP）**：
1. **启动页**：显示服务端连接状态（未连接 / 连接中 / 已就绪）。
2. **设置页**：
   - 输入 AI 模型 API Key、模型 URL、模型名称。
   - 调用 `POST /api/v1/settings` 保存。
   - 验证连通性（调用 `/health` 或发送测试请求）。
3. **配置建议页**：
   - 输入用户画像（风险偏好、投资期限、月现金流、应急金月数）与当前持仓。
   - 调用 `POST /api/v1/plan`。
   - 展示 `proposed_portfolio`（建议权重表）、`risk_report`（风险指标与触发规则）、`rationale`（理由与引用）、`rebalancing_actions`（操作清单）。
4. **流式问答页**：
   - 输入 prompt。
   - 调用 `POST /api/v1/chat/stream`（SSE）。
   - 实时渲染 token 流；展示 cite / warning / metric 事件。

**验收标准**：
- 四个页面均可访问且功能正常。
- SSE 流式输出实时渲染，无丢帧、无阻塞。
- 设置页写入 API Key 后，配置建议和问答功能可正常使用。
- 响应式布局，在 1280×720+ 分辨率下可用。

### 11.3 第八阶段：桌面壳与本地启动器（Tauri）

**目标**：将 Web 前端 + finguard.exe 打包为 Windows 桌面应用，实现一键启动。

**技术栈**：Tauri v2（Rust + WebView2）。

**核心功能**：
1. **Sidecar 进程管理**：
   - finguard.exe 作为 Tauri sidecar 声明。
   - 应用启动时自动拉起 finguard.exe。
   - 轮询 `/health` 确认服务就绪后加载前端。
   - 应用关闭时自动终止 finguard.exe。
2. **状态展示**：
   - 标题栏 / 状态栏显示服务状态（启动中 / 运行中 / 异常）。
   - 启动失败时显示错误信息与日志路径。
3. **托盘图标**（可选）：最小化到系统托盘，后台运行。
4. **配置持久化**：用户配置（API Key、URL）保存到用户目录（`%APPDATA%/finguard/`），应用重启后自动加载。

**验收标准**：
- 双击启动应用 → finguard.exe 自动启动 → 前端就绪 → 可正常使用所有功能。
- 关闭应用 → finguard.exe 自动退出，无残留进程。
- 首次启动自动引导到设置页填写 API Key。

### 11.4 第九阶段：打包分发与最终交付

**目标**：生成可分发的安装包，面向普通用户可直接安装使用。

**交付物**：
1. **安装包**：
   - Windows 安装包（NSIS `.exe` 或 MSI）。
   - 或便携版 `.zip`（解压即用）。
   - 内含：桌面应用（含前端资源）、finguard.exe、默认配置文件、`config/rules.yaml`。
2. **首次使用引导**：
   - 安装后首次启动 → 引导页面 → 填入 API Key 和模型 URL → 验证连通 → 进入主界面。
3. **用户手册**：
   - 安装说明、首次配置、功能使用说明。
   - 支持的 AI 模型列表（OpenAI、DeepSeek、Qwen 等）与配置示例。
   - 常见问题与故障排查。
4. **端到端验收**：
   - 在干净 Windows 10/11 系统上安装 → 首次配置 → 获取投资建议 → 流式问答，全流程畅通。
   - 安装包体积 ≤ 30 MB。

**验收标准**：
- 面向非技术用户可在 3 分钟内完成安装与首次配置。
- 全功能可用：配置建议、流式问答、风控报告。
- 无需安装额外依赖（WebView2 Runtime 随 Windows 10/11 内置）。

### 11.5 第十阶段：Value Cell 量化价值分析系统

**目标**：将 FinGuard 从通用 LLM 网关转型为"定性 + 定量"的量化价值投资分析系统，放弃复杂且不稳定的财报 PDF RAG 路径，改用结构化数据 + 搜索引擎方案。

**核心设计思路**：
- **定量 (Quantitative)**：通过 Yahoo Finance API 获取结构化财务指标（PE、PB、PEG、未来预期增长率），计算历史 PE 分位与安全边际。
- **定性 (Qualitative)**：通过搜索引擎 API 获取目标公司的护城河、管理层诚信与核心商业模式信息，由 LLM 进行定性评分。

**新增模块**：
1. **Yahoo Finance 数据客户端** (`valuation/yahoo_finance_client`)：
   - 异步 HTTP 请求 Yahoo Finance API，获取实时与历史财务指标。
   - 返回结构化数据：当前 PE / PB / PEG / 未来预期增长率 / 5 年 PE 历史序列。
   - 内存中计算，不存储中间状态。
2. **搜索引擎 Wrapper** (`valuation/search_wrapper`)：
   - 调用搜索引擎 API（Google Custom Search / Bing / SerpAPI）。
   - 获取目标公司护城河评价、管理层信息、商业模式分析等定性内容。
   - 将搜索结果摘要传递给 LLM 进行定性评分（0-10）。
3. **安全边际计算** (`valuation/safety_margin`)：
   - `calculateSafetyMargin()`：对比当前 PE 与过去 5 年历史均值。
   - 评价逻辑：若 (PE < 历史均值) AND (PEG < 1.0)，判定为"安全边际区间"。
   - 所有金融数据计算在内存中完成，不持久化中间结果。

**新增接口**：
- `POST /api/v1/valuecell` — 输入股票代码（如 `AAPL`），返回 Value Cell 分析报告。

**输出格式（Markdown 结构化）**：
```markdown
## Value Cell 分析报告 — {symbol}

| 指标 | 值 |
|------|----|
| 定性评分 | 7.5 / 10 |
| 当前 PE | 18.2 |
| 5年 PE 均值 | 22.4 |
| 当前 PEG | 0.85 |
| 安全边际判定 | Yes ✅ |

### 核心投资结论
该公司当前估值低于历史均值，PEG < 1 表明增长被低估，处于安全边际区间。

### 定性分析
- 护城河：...
- 管理层：...
- 商业模式：...
```

**技术要求**：
- 语言：C++17，所有 API 调用异步（Drogon HttpClient）。
- 安全：金融计算纯内存，不写中间文件。
- 新增源文件注册到 `CMakeLists.txt`。

**前端扩展**：
- 新增 Value Cell 分析页面：输入股票代码 → 调用 `/api/v1/valuecell` → 渲染 Markdown 报告。

**验收标准**：
- 输入任意美股代码（如 AAPL、MSFT），返回完整 Value Cell 报告。
- 报告包含：定性评分 (0-10)、当前 PEG、安全边际判定 (Yes/No)、核心投资结论。
- API 响应时间 < 5s（含外部 API 调用）。
- 外部 API 失败时熔断降级，返回可用的部分报告。

## 12. 风险与对策
- 数据源不稳定：启用熔断与降级数据源。
- 模型成本波动：多模型 fallback，按风险与场景选择。
- 规则误判：规则配置化 + 灰度阈值策略。
- 性能瓶颈：压测驱动扩容与缓存策略优化。
- Tauri / WebView2 兼容性：Windows 10 1803+ 均支持 WebView2；Windows 7 用户需提示手动安装 WebView2 Runtime。
- Rust 工具链：Phase 8 开发需安装 Rust 工具链，但 Rust 侧代码量较小（主要是进程管理与配置读写胶水代码）。
- Yahoo Finance API 变更/限流：做好 API 降级与缓存，必要时引入备用数据源（如 Alpha Vantage）。
- 搜索引擎 API 配额与成本：按需选择免费/付费方案，优先 Google Custom Search（免费 100 次/天）或 SerpAPI。
- 定性评分一致性：LLM 打分可能波动，需设置 temperature = 0 并固定 prompt 模板，必要时引入多次采样取均值。
