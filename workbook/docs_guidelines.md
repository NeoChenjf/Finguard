# 文档编写、教学与格式规范

本文件规定项目内所有文档（规则文档、教学文档、阶段文档）的结构、格式、编辑规则与质量要求。

---

## 1. 元数据

所有 `workbook/*.md` 文件顶部必须包含 YAML 元数据块：

```yaml
---
title: 文档标题
tags: [tag1, tag2]
last_updated: YYYY-MM-DD
source: workbook
status: draft | published
---
```

## 2. 格式与编码

- 中文文档面向实际使用者，句子短、术语少；出现术语需附白话解释。
- 中文文档保存为 UTF-8（建议带 BOM 以避免部分编辑器乱码）。
- 命令和代码片段必须可直接复制到终端或编辑器复现（必要时说明执行目录和环境）。

## 3. 教学文档标准

当用户要求"教学/讲解/学习路线/教程"时，生成独立 `.md` 教学文档，包含以下结构：

1. **标题与目标**（一句话说明学习目标）
2. **学习路线/分层**（分阶段，从易到难）
3. **核心概念解释**（至少一个生活类比，如"做饭"或"流水线"）
4. **关键流程/步骤**（步骤化、可复刻）
5. **真实代码片段**（必须逐行注释）
6. **小结 + 自测题**（自测题须附答案）
7. **下一步建议**（后续深入方向）

### 教学问答

当用户进行教学问答（术语解释、概念澄清）时，也遵循教学规范：先给目标与白话解释，再给例子或类比，最后给小结与自测题（含答案）。

## 4. 文档编辑防重复规则

对文档进行结构重构或大范围改写时：

- **先删后写**（delete-then-create）：禁止仅在头部/尾部追加新版本而不删除旧版本。
- 若重构涉及全文，优先"删除文件 → 重建文件"。
- 重构完成后，验证文件中不存在重复的顶级标题。

验证命令：
```powershell
(Get-Content "target.md" | Select-String "^# ").Count   # 应为 1
(Get-Content "target.md").Count                          # 行数应合理
```

## 5. 临时文件生命周期

阶段性临时文件（completion report、中间摘要、验收草稿）在内容被合并到主文档后，应及时删除，避免信息源分裂。

- 合并完成后确认主文档包含所有必要内容。
- 删除临时文件并在变更日志中注明来源。

## 6. 变更日志

自动加入或修改规则时，必须在文档末尾追加变更日志：

```
[YYYY-MM-DD] (自动加入|手动更新|需复核) 来源摘要
```

## 7. 审阅与质量门槛

重要或高风险文档（含系统命令、权限变更、网络调用）必须经人工审阅后才标注 `status: published`。
