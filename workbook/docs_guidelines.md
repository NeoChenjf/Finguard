# 文档与教学规范（Docs & Tutorials）

## 目的

- 规范项目内教学文档与规则文档的结构和质量，确保后续 agent 与开发者能快速理解并复现操作。

## 文档结构要求

- 每个教学/规则文档必须包含以下部分：
  1. 标题与目标（一句话说明学习目标）
  2. 学习路线/分层（分阶段，从易到难）
  3. 核心概念解释（至少一个生活类比）
  4. 关键流程/步骤（步骤化、可复制）
  5. 真实代码片段或命令示例
  6. 小结 + 自测问题（可用于验收）
  7. 下一步建议（后续深入方向）

## 元数据与文件头

- 所有 `workbook/*.md` 文件顶部必须包含 YAML 元数据块，至少包含：`title`、`tags`、`last_updated`、`source`、`status`。

## 格式与语言

- 中文文档面向小白，句子短、术语少，出现术语需附白话解释。
- 中文文档保存为 UTF-8 编码（建议带 BOM 以避免部分编辑器乱码）。

## 示例与可复现性

- 所有命令和代码片段必须可直接复制到终端或编辑器并在提供的步骤中复现（必要时说明执行目录和环境）。

## 变更与审计

- 自动加入或修改规则时，必须在文档末尾追加变更日志，格式：`[YYYY-MM-DD] (自动加入|手动更新|需复核) 来源摘要`。
- 对于语义相似但不确定的变更，应标注为“需复核”。

## 审阅与质量门槛

- 重要或高风险的文档（如包含系统命令、权限变更、网络调用）必须经人工审阅后才标注为 `status: published`。

## 示例生活类比（模板）

- 若需解释复杂概念，使用“做饭”或“流水线”类比：例如把请求-处理-返回比作“点菜→厨房做菜→上菜”的流程。

## 教学文档标准（补充）

- 当用户要求“教学/讲解/学习路线/教程”时，必须生成独立 `.md` 教学文档，包含：标题与目标、学习路线/分层、核心概念解释、关键流程/步骤、真实代码片段、小结 + 自测题、下一步建议。
- 表述面向小白，短句少术语；出现术语须有白话解释。
- 当用户说“教教我”时，默认更新当前教学文档（当前为 `学习文档/20260202/study.md`），除非用户明确要求新建文档。

## 产品定位与文风

- 输出文档默认面向上架产品与真实用户，不使用面试/简历语气；如需面试版须用户明确说明。

## 文件编码

- 中文文档使用 UTF-8（带 BOM）以避免乱码。
