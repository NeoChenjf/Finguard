# Phase Fundamental-Analysis-Optimization — 基本面分析功能优化

## Chapter 1：Spec 前置计划（冻结）

### 1) 页面结构冻结：恢复单仪表盘
- **决策/推荐**：ValueCell 页面恢复为最初那种**单仪表盘**主视图，只保留必要指标与规则反馈。
- **理由**：这轮需求只是调整 PEG、ROE、星级规则与少量展示，不是重做信息架构；多仪表盘、多模块会稀释核心信息。
- **详情**：
  - 保留：输入框、快捷样本、单个主仪表盘、规则列表、告警。
  - 删除：`完整分析报告` 模块。
  - 主页面不再出现多套评分面板或多层兼容展示区。

### 2) 数据口径冻结：上游提供什么、前端展示什么
- **决策/推荐**：本轮主口径按你最新确认冻结为：
  - 上游（Yahoo / Alpha）主供给字段：
    - 过去 8 年净利润（用于 growth）
    - 当前股价 `price`
    - 当前 PE `current_pe/trailing_pe`
    - 过去 5 年平均 ROE `avg_roe_5y`
    - 负债率 `debt_ratio`
    - PB `price_to_book`
  - 本地派生字段：
    - `growth_5y_cagr`
    - `peg`
- **理由**：必须统一成“上游原始字段 + 本地少量派生”的模型，避免前后端各自拼公式导致串线。
- **详情**：
  - `PE` 主展示优先使用上游 `current_pe/trailing_pe`。
  - 只有当上游 `current_pe/trailing_pe` 缺失时，才允许后端用 `price / eps` 作为兼容回退。
  - `PEG` 不再信任旧兼容 `current_peg` 作主展示，必须按：
    - `PEG = PE / (growth_5y_cagr * 100)`
  - `ROE` 主展示使用上游直接字段 `avg_roe_5y`，不再由前端硬算，也不再要求后端为了展示再拼 5 年股东权益历史。
  - `负债率` 口径冻结为：
    - `debt_ratio = total_liabilities / total_assets`

### 3) 核心计算口径冻结
- **决策/推荐**：growth 继续只采用“最近 3 年均值 vs 5 年前 3 年均值”的单一口径。
- **理由**：这是当前最稳定、最容易手算复核的增长口径。
- **详情（公式）**：
  - `np_recent3_avg = Avg(NP[t-2], NP[t-1], NP[t])`
  - `np_past3_avg = Avg(NP[t-7], NP[t-6], NP[t-5])`
  - `growth_5y_cagr = (np_recent3_avg / np_past3_avg)^(1/5) - 1`
  - `PE = current_pe（优先）`，否则 `price / eps`
  - `PEG = PE / (growth_5y_cagr * 100)`

### 4) 五星规则冻结
- **决策/推荐**：单仪表盘只保留 5 条规则，每条最多 1 星。
- **理由**：用户已经明确希望保留原来好用的仪表盘体验，只调整规则，不再引入其它复杂评分项。
- **详情**：
  1. `avg_roe_5y > 20%`：得 1 星
  2. `PEG < 1`：得 1 星
     - `<1` = 非常安全
     - `1~2` = 正常（观察，不得星）
     - `>2` = 充分价格反应
  3. `debt_ratio <= 50%`：得 1 星
  4. `PE * PB < 2.25`：得 1 星
  5. `growth_5y_cagr > 20%`：得 1 星
- **展示要求**：
  - 得星项显示绿色
  - `PEG 1~2` 显示黄色“正常/观察”
  - 未通过显示红色
  - 缺失显示灰色 `N/A`

### 5) 告警与降级规则冻结
- **决策/推荐**：告警不阻断响应，但必须在主页面可见。
- **详情**：
  - 保留/新增 warnings：
    - `invalid_eps`
    - `insufficient_net_profit_history`
    - `invalid_growth_base`
    - `non_positive_growth`
    - `low_growth_5y_cagr`
    - `high_debt_ratio`
    - `missing_debt_ratio`
    - `missing_roe_history`
  - 主卡片**不允许**再回退到旧兼容 `current_peg` 造成“新 growth + 旧 PEG”混算。
  - 当主口径字段缺失时，页面主展示直接显示 `N/A`。

## Chapter 3：任务执行详情

### 步骤执行记录

#### Step 1：问题定位
- 已确认最初页面异常的核心根因有两类：
  1. **PEG 串线**：页面曾出现“growth 用新口径、PEG 却回退旧 `current_peg`”的问题。
  2. **ROE 失真**：前端曾本地拼公式计算 ROE，导致 251% 这类明显失真数值。
- 页面结构也发生了需求漂移：从最初单仪表盘变成了多块面板，偏离了你的真实需求。

#### Step 2：后端口径收敛
- 已修改 `finguard/src/valuation/yahoo_finance_client.cpp`：
  - `metrics.pe` 优先使用上游 `trailing_pe/current_pe`。
  - `metrics.peg` 统一使用 `PE / (growth_5y_cagr * 100)` 重算。
  - `avg_roe_5y` 改为优先使用上游直接字段。
  - 不再为了主展示继续抓取并拼接 5 年股东权益历史来计算 ROE，减少无效复杂度与运行时负担。
  - 新增 ROE 百分数归一化处理：若上游返回 `25.1` 这类百分数写法，会规范成 `0.251`。
  - 当 Yahoo crumb 不可用时，直接跳过 Yahoo 主链并尽快降级到 Alpha，避免“Yahoo 明明失败却卡很久”。

- 保留负债率主口径：`总负债 / 总资产`。
- 边界告警继续保留：高负债率、低 growth、缺失 ROE、缺失 debt ratio 等。

### 问题与修复

#### 问题 1：TCOM 页面 PEG 对不上 `7.42 / 22.53`
- **现象**：页面曾显示 `PEG = 1.91`，但按 `7.42 / 22.53` 应接近 `0.33`。
- **原因**：旧版主卡片把新 growth 与旧兼容 `current_peg` 混在一起展示。
- **修复**：主卡片 PEG 改为统一使用 `PE / (growth_5y_cagr * 100)` 重算，不再回退旧 `current_peg`。

#### 问题 2：5 年平均 ROE 出现 251%
- **现象**：页面曾展示明显失真的超高 ROE。
- **原因**：前端曾本地伪造 ROE 计算逻辑。
- **修复**：页面只展示上游 `avg_roe_5y`；后端仅做格式归一化，不再本地硬算。

#### 问题 3：页面从一个好用的仪表盘变成多个面板
- **现象**：视觉结构偏离初始版本，信息层次变乱。
- **原因**：在中途口径调整时把页面做成了“多区块收敛页”。
- **修复**：回归单仪表盘，只保留一个主面板 + 规则 + 告警。

#### 问题 4：页面长时间无结果，看起来像“死掉”
- **现象**：输入 TCOM / PDD 后长时间不返回，像卡死。
- **原因**：
  1. ValueCell 路由还在等待搜索/LLM 链路；
  2. Yahoo crumb 不可用时没有足够快地跳过 Yahoo；
  3. Alpha 达到日额度时没有给出明确解释。
- **修复**：
  - ValueCell 默认不再阻塞等待定性搜索与 LLM 报告；
  - Yahoo crumb 不可用时直接走降级；
  - Alpha 额度耗尽时返回明确 warning，前端可读展示。

#### 问题 5：输入 `BRK` 时大量字段缺失
- **现象**：按你指定的测试样本输入 `BRK`，页面会出现大面积 `N/A`，并伴随明显更长的等待时间。
- **原因**：数据源实际更稳定识别的是 `BRK-B`，而不是裸 `BRK`；此前后端没有做 symbol alias 归一。
- **修复**：后端新增 symbol 规范化，`BRK / BRKB / BRK.B / BRK/B` 一律映射到 `BRK-B`，并在响应中保留 `requested_symbol` 方便核对。

#### 问题 6：Finnhub 指标接口首次成功后没有复用，导致重复查询仍然很慢
- **现象**：同一股票即使已经成功查过一次，第二次仍会重新等待 Finnhub 大包。
- **原因**：此前只缓存了净利润历史，没有缓存主卡片指标。
- **修复**：
  - 新增 `uploads/tmp/valuation_quote_metrics_<symbol>.json`
  - 缓存字段：`PE / PB / avg_roe_5y / EPS / BVPS`
  - 后续请求直接复用缓存指标，只保留 live quote 刷新当前价格
  - 前端新增 `finnhub_metric_cache_fallback` 告警文案

### 本轮关键验证点

#### TCOM（规则期望）
- 若 `PE = 7.42`
- 若 `growth_5y_cagr = 22.53%`
- 则页面主展示 PEG 应接近：
  - `7.42 / 22.53 = 0.3293`
- 结论：
  - 这是 `PEG < 1`
  - 应判定为“非常安全”
  - 应得 PEG 这一颗星

#### PEG 评分冻结
- `<1`：通过 / 得星
- `1~2`：正常 / 观察 / 不得星
- `>2`：充分价格反应 / 不得星
