## 1. CMake（完整章）

- 核心作用：CMake 是一个跨平台的“生成器”（generator），把 `CMakeLists.txt` 中的高层构建声明翻译成平台/编译器特定的构建脚本（Makefile、Ninja、MSBuild 等）。

- 重要概念：
  - `project()`：定义工程名与语言。
  - `add_executable()` / `add_library()`：声明目标（target）。
  - `target_include_directories()` / `target_link_libraries()`：指定头路径与链接依赖。
  - 配置类型：Debug / Release。
  - out-of-source build：把构建目录放到 `build/`，保持源码干净。

- 最小示例：

  - 目录：

    - CMakeLists.txt
    - src/main.cpp

  - `CMakeLists.txt`:

```cmake
cmake_minimum_required(VERSION 3.15)
project(minapp LANGUAGES CXX)
add_executable(minapp src/main.cpp)
set_target_properties(minapp PROPERTIES CXX_STANDARD 17)
```

  - `src/main.cpp`:

```cpp
#include <iostream>
int main(){ std::cout<<"hello"<<std::endl; return 0; }
```

- 构建命令（推荐 out-of-source）：

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

- 常见问题与调试：
  - 找不到库：确认 `find_package` 或 `vcpkg` toolchain 是否传递到 CMake（见 vcpkg 章）。
  - 头文件找不到：使用 `target_include_directories` 或 `target_link_libraries`（目标接口包含）。
  - 多配置生成器（Visual Studio）：使用 `cmake --build . --config Release`。

## 2. Drogon（完整章）

- 是什么：高性能的 C++ Web 框架，异步 IO，适合写 HTTP 服务、WebSocket 等。

- 入门要点：
  - 路由（route）映射 URL 到 handler。
  - handler 接收请求并返回响应。
  - 支持异步与同步 handler。

- 最小示例（依赖已通过 vcpkg 安装 drogon）：

  - `src/main.cpp`:

```cpp
#include <drogon/drogon.h>
using namespace drogon;
int main(){
  app().registerHandler("/ping", [](const HttpRequestPtr &req, std::function<void (const HttpResponsePtr &)> &&callback){
    auto resp = HttpResponse::newHttpJsonResponse("");
    resp->setBody("{\"status\":\"ok\"}");
    callback(resp);
  });
  app().run();
}
```

  - CMake 示例（链接 drogon）：

```cmake
find_package(Drogon CONFIG REQUIRED)
add_executable(myserver src/main.cpp)
target_link_libraries(myserver PRIVATE Drogon::Drogon)
```

- 运行后测试：

```bash
curl http://localhost:8848/ping
```

- 常见问题：
  - 端口冲突：检查配置文件（drogon.json）或命令行参数。
  - 依赖缺失：通过 `vcpkg install drogon` 或源码编译安装。

## 3. spdlog（完整章）

- 简介：高性能 C++ 日志库，易用，零配置即可使用。

- 最小示例：

```cpp
#include <spdlog/spdlog.h>
int main(){
  spdlog::info("hello {}", "world");
  spdlog::warn("a warning");
}
```

- 常用配置：
  - 设置等级：`spdlog::set_level(spdlog::level::info);`
  - 输出到文件：`auto logger = spdlog::basic_logger_mt("file_logger", "logs/log.txt");`

- 在 CMake 中链接：`target_link_libraries(myexe PRIVATE spdlog::spdlog)`（通过 vcpkg 或 find_package）。

## 4. nlohmann/json（完整章）

- 简介：单头文件 JSON 库，API 直观，操作像 Python 的 dict。

- 示例：

```cpp
#include <nlohmann/json.hpp>
using json = nlohmann::json;
json j = { {"name","alice"}, {"age",30} };
std::string s = j.dump();
auto name = j["name"].get<std::string>();
```

- 注意：访问不存在的键会抛异常或产生未定义行为，推荐先 `contains()` 或使用 `value()` 提供默认。

## 5. Google Test（gtest，完整章）

- 简介：常用 C++ 单元测试框架，断言丰富。

- 最小示例：

```cpp
#include <gtest/gtest.h>
TEST(MathTest, Add){ EXPECT_EQ(2+2, 4); }
int main(int argc, char **argv){ ::testing::InitGoogleTest(&argc, argv); return RUN_ALL_TESTS(); }
```

- CMake 集成：

```cmake
find_package(GTest CONFIG REQUIRED)
add_executable(tests test/main.cpp)
target_link_libraries(tests PRIVATE GTest::gtest_main)
enable_testing()
add_test(NAME all_tests COMMAND tests)
```

## 7. curl 与调试（完整章）

- curl 是命令行 HTTP 客户端，常用于手动测试与脚本化检查。

- 常用命令：

```bash
# 简单 GET
curl http://localhost:8848/ping
# POST JSON
curl -X POST -H "Content-Type: application/json" -d '{"name":"x"}' http://localhost:8848/api
# 查看响应头
curl -i http://localhost:8848
# 保存响应到文件
curl http://example.com -o out.html
```

- 调试技巧：加 `-v` 查看细节，`--trace-ascii` 打印完整请求/响应。

## 9. CMake vs 构建工具（补充）

- 区别与责任划分：
  - CMake：负责描述“我需要哪些目标、依赖和如何链接”，并生成平台构建脚本。
  - 构建工具（make/ninja/MSBuild）：负责实际执行编译与链接步骤。

- 小贴士：调试构建问题时，先看 CMake 配置输出（`cmake ..` 中的 summary），再查看生成构建文件（如 `build/Makefile` 或 `build/build.ninja`）。

## 10. vcpkg（完整章）

- 作用：跨平台 C++ 包管理器，便于安装并在 CMake 中消费第三方库。

- 快速安装（Windows）：

```powershell
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
.\bootstrap-vcpkg.bat
```

- 常用命令：

```powershell
vcpkg install spdlog nlohmann-json drogon gtest
# 使用 manifest（项目内的 vcpkg.json）
vcpkg install --manifest
```

- 在 CMake 中使用：

```bash
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
```

- 注意：如果使用 manifest 模式，项目依赖会记录在 `vcpkg.json`，团队成员只需运行 `vcpkg install --manifest`。

## 附：常见问题速查（FAQ）

- 链接失败：确认在 CMake 配置时传入了 `-DCMAKE_TOOLCHAIN_FILE`（vcpkg）或正确的 `find_package`。
- 运行时缺少 DLL（Windows）：确保 vcpkg 的运行时库在 PATH，或使用静态链接。
- Drogon 端口未启动：检查配置文件 `drogon.json` 与应用日志输出。
