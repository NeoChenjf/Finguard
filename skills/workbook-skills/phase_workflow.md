# 阶段流程与验收（Phase Workflow）

阶段文档要求
- 当用户提出“开始阶段任务/阶段推进/进入某阶段”时，创建或更新 `phase{N}.md`，包含：
  1) 阶段总览  2) Todo List（已完成/未完成）  3) 阶段工作日志（问题与解决过程）

验收要求
- 每阶段完成后执行验收检查（自动化优先），并将验收结果写入 `phase{N}.md`。

安装与记录
- 所有依赖/工具安装步骤必须记录在 `phase{N}.md`，含命令、结果、失败原因与教学性步骤，确保可复现。

任务交接
- 交接需更新 `next_plan.md`，写清当前状态、阶段目标、任务清单、验收标准、未决问题与参考文件。
