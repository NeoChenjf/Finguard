# 自动化操作（Automation）

概要
- 自动化操作优先目标：可复现、可回滚、可追溯。
- 自动化须在明确目录/环境下运行，避免路径或虚拟环境导致导入/启动失败。

自动启动后端
- 在检测到后端未运行时，优先尝试以下步骤（无需额外确认，但需记录日志）：
  1. 进入 `backend` 目录
  2. 激活虚拟环境并以可重载模式启动服务：
     ```powershell
     cd backend
     & ..\.venv\Scripts\Activate.ps1
     python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
     ```
  3. 如果启动失败，自动收集常见错误信息并提示修复建议（如 app 未定义、端口占用、环境变量缺失等）。

失败处理与提示
- 自动化失败时输出可操作的诊断步骤和可能的修复命令，避免直接修改用户敏感配置。
- 对无法安全自动修复的问题，停止自动化并向用户报告详细日志与建议步骤。

日志与可追溯性
- 每次自动化操作都应写入日志（包含命令、时间、结果、输出摘要，敏感信息脱敏）。

风控与权限
- 用户应明确授权后再允许自动化操作；授权可在会话中撤销。
- 禁止自动执行高风险系统命令或从不可信来源下载并执行代码（详见 `safety.md`）。
