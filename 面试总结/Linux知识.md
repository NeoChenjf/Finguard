**开发 — 常见命令与面试要点**
- **`curl`**: 用处：接口测试、SSE 测试、降级 HTTP 客户端（项目中大量使用 `curl.exe` 与 `--config`/`-K` 规避转义）。
  - 常考点：参数含义（`-X`、`-H`、`-d`、`-s`、`-v`、`-N`、`-L`、`-4`、`--resolve`、`--proxy`、`--max-time`、`-w`、`-o`）；SSE 场景下 `-N` 的作用；`--config/-K` 为何能规避命令行转义；`-s -o /dev/null -w "%{http_code}\n"` 模拟探活的原理。
  - 深入点：`curl` 与系统网络栈差异（为什么能避开某些 DNS/IPv6 问题）；子进程隔离（`fork+exec` 调用外部 curl）代价与风险（阻塞、僵尸进程、超时回收）。
  - 面试问法示例：如何用 `curl` 发起 SSE 请求？`curl` 在高并发下作降级的利弊？如何用 `curl` 强制 IPv4？

- **`cmake` / `cmake --build`**: 用处：项目构建（生成构建目录、跨平台构建描述）。
  - 常考点：`-S . -B build` 含义、`-DCMAKE_TOOLCHAIN_FILE` 的用途（vcpkg 集成）、`--config Debug|Release` 的差别、`-DCMAKE_BUILD_TYPE`（单目录 Unix 用法）。
  - 深入点：CMake 并不直接编译而是生成平台原生工程（Makefile / Ninja / Visual Studio）；如何为 CI 优化构建（并行、增量、目标拆分）。
  - 面试问法示例：CMake 配置流程与构建流程有何不同？如何在 CI 中做 Release 构建并保证产物可复现？

- **`ctest`**: 用处：运行 gtest 单元测试（项目中用 `ctest --output-on-failure`）。
  - 常考点：如何在不同配置下运行测试、如何收集失败输出、并行/选择性执行特定测试。
  - 面试问法示例：如何把单元测试接入 CI 并在失败时收集诊断信息？

- **`ss` / `netstat`**: 用处：查看端口占用与 socket 状态（如 `ss -lntp | grep 8080`、`netstat -anp | grep 8080`）。
  - 常考点：`ss` 与 `netstat` 区别（`ss` 更快、直接读取内核），常用选项含义（`-l`、`-n`、`-t`、`-p`）；常见 socket 状态（`TIME_WAIT`、`CLOSE_WAIT`）及其含义。
  - 深入点：为何大量 `TIME_WAIT` 会影响可用端口？如何在应用层/内核层缓解（复用端口、短连接与长连接设计）。
  - 面试问法示例：如何定位 8080 被哪个进程占用？解释 `TIME_WAIT` 的产生与处理策略。

- **`ping`**: 用处：基础网络连通性检查（文档中作为排查网络层的第一步）。
  - 常考点：ICMP 与 TCP 层面的差别；使用 `-c` `-w` 等控制包数/超时。
  - 面试问法示例：`ping` 可否完全替代 TCP 层连通性检测？为什么？

- **`grep` / `awk` / `sed` / `find` / `xargs`**: 用处：日志过滤、批量查找、文本处理（文档中多处示例/建议）。
  - 常考点：正则与基本选项；`grep -R`、`awk` 字段处理、`sed` 的替换用法、`find` 与 `xargs` 的高效组合。
  - 面试问法示例：如何用一行命令查找并替换目录下所有文件中的字符串？`xargs -P` 并行执行的风险是什么？

**部署 — 常见命令与面试要点**
- **`docker` / `docker compose up`**: 用处：容器化一键启动（项目书提到容器化与 `docker compose up`）。
  - 常考点：`docker build` 与 `docker run` 参数（端口映射 `-p`、环境变量 `-e`、卷挂载 `-v`）、Compose 文件作用、服务依赖与健康检查。
  - 深入点：容器网络模式（bridge/host/overlay）、镜像体积优化（多阶段构建）、安全面（运行用户、只读根文件系统）。
  - 面试问法示例：如何为一个高并发服务设计 Docker Compose？如何在生产环境确保容器重启策略与资源限制？

- （项目中以 Windows 为主，但部署常见的 Linux 实践）
  - **打包/压缩**（若出现）：`tar`/`gzip` 常用于发布包；面试常考 `tar` 常用选项与流式压缩/解压原理。

**运维 — 常见命令与面试要点**
- **系统与服务管理**（项目未直接使用 `systemctl`，但面试时常被问及）
  - 常考点：`systemctl start|stop|status|enable` 含义；`journalctl` 日志查询（`-u 服务`、`-f` 实时跟随）；如何将服务转为 systemd 服务单元。
  - 面试问法示例：如何把一个自启动程序包装成 systemd 服务？如何排查启动失败的日志？

- **进程管理与信号**（文档中提及 `fork`/`exec`/`waitpid` 的设计思路）
  - 常考点：`ps aux`、`top`/`htop` 查看进程与资源；`kill` 信号语义（`SIGTERM` / `SIGKILL`）；僵尸进程与 `waitpid` 的关系。
  - 深入点：为什么 `fork+exec` 会导致较高开销？如何通过非阻塞 `waitpid` 或子进程回收避免僵尸？面试可能追问系统调用开销与替代方案（线程/异步库）。
  - 面试问法示例：解释僵尸进程产生原因和如何在服务设计中避免它们。

- **网络排查与代理**
  - **常用命令**：`curl`（探活、代理测试）、`ss`/`netstat`（socket 状态）、`traceroute`/`tracepath`（路由追踪，若需要）。
  - 常考点：代理（`--proxy`）、DNS 解析差异（系统解析 vs 库解析如 c-ares）、IPv4/IPv6 优先级及其排查方法。
  - 面试问法示例：如何定位某台机器访问外部 API 失败，是 DNS、路由还是防火墙？

**每个命令面试复习要点（速查清单）**
- `curl`: 熟练项：请求方法、头、数据、SSE(`-N`)、配置文件(`-K`)、超时与重试、HTTP 返回码提取(`-w`)、与系统网络栈的关系、作为降级子进程的优缺点与超时回收策略。
- `cmake`/`ctest`: 熟练项：配置 vs 构建阶段、工具链文件、构建类型、测试运行/报告接入 CI 的方法。
- `ss`/`netstat`: 熟练项：常用选项、解析 socket 状态、发现端口占用的排查流程与 root 权限问题。
- `ps`/`top`/`kill`: 熟练项：查看 CPU/内存消耗、发送信号、如何安全回收子进程。
- `git`（隐含）: 虽未特别列出，但开发流程中常考 `git` 基础操作、冲突解决、rebase vs merge 的面试题。
- `docker`/`docker-compose`: 熟练项：镜像构建优化、Compose 服务依赖、健康检查与资源限制、生产环境的替代（Kubernetes 简述）。

**项目中出现的典型面试问答示例（备考）**
- 问：项目中为何使用 `curl` 作为降级方案？答要点：进程隔离、成熟的网络兼容性，代价为系统调用/子进程开销及需妥善回收（避免僵尸）；面试官可能继续问如何在高并发下改进（连接池、异步 HTTP 客户端、限流）。
- 问：如何用命令行验证 `/health` 接口的可用性并把结果作为 CI 探针？答要点：`curl -s -o /dev/null -w "%{http_code}\n" http://host/health`，结合退出码判断并在 CI 中失败。可拓展：使用 `jq` 解析 JSON 响应。
- 问：`ss -lntp | grep 8080` 返回很多 `TIME_WAIT`，说明什么？如何排查？答要点：说明短连接或高并发导致大量短连接关闭，检查服务是否使用连接复用、调整 TCP TIME_WAIT 回收或启用 socket 选项（如 `SO_REUSEADDR`）并评估影响。

**结论与学习建议（面试优先级）**
- 优先掌握（高频面试考点）：`curl` （各种参数与网络诊断）、`cmake`/`ctest`（构建与测试流程）、`ss`/`netstat`（网络排查）、进程管理（`fork`/`exec`/`waitpid` 基本原理与僵尸进程处理）。
- 次要掌握（加分项）：`docker`/`docker-compose`（容器化部署基本操作）、`grep`/`awk`/`sed`（日志与文本处理）、`tar`/`gzip`（打包发布）。
