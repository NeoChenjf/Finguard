# 2026-02-04 学习总览：基本面模板与响应侧规则

## 标题与目标

读完这份文档，你能：
- 明确 LLM 基本面分析的强制模板与章节列表（顺序不强制）。
- 理解 check_response 要检查哪些内容，以及为什么要这样检查。
- 说清楚 warning 字段的意义与使用场景。

## 学习路线 / 分层

1. 先理解“response 是什么”（LLM 只做基本面分析文本）。
2. 掌握强制模板与各章节要点。
3. 学会响应侧规则（check_response）如何产生 warning。
4. 记住阈值判断与语义判断的合规要求。

## 核心概念解释（含类比）

- 强制模板：给 LLM 的“标准答题卡”。
  - 类比：考试答题纸固定格式，少一栏就算未答。
- check_response：对“答案内容”做合规与完整性检查的规则引擎。
  - 类比：阅卷老师，负责查漏补缺与扣分提示。
- warning：把问题显性化的提示对象。
  - 类比：交通黄灯，提醒你需要谨慎或补充信息。

## warning 字段含义（生活化例子）

- code：规则编号，机器识别用。  
  - 类比：超市货架的“条形码”，电脑扫描就知道是哪一类问题。
- section：问题发生的章节位置。  
  - 类比：试卷上的“第几题”，让你知道错在哪一题。
- reason：具体原因说明，给人读的。  
  - 类比：老师的评语，比如“少了步骤 2”。
- trigger_terms：触发关键词或数值，固定为数组（例如：`["负债率 65%"]`）；无触发词时使用空数组 `[]`。  
  - 类比：交通抓拍里显示的“车速 98km/h”。
- basis：触发依据（阈值/理论来源/模板要求），固定格式：`阈值=<...>；来源=<...>`（非数值类写语义阈值，例如：`阈值=必须包含合规声明`）。  
  - 类比：交规里写的“限速 60”，说明你为何被判超速。
- evidence：原文片段字符串（必填）。  
  - 类比：摄像头拍到的画面，便于复核是否真的违规。
- 缺失类 warning 的 evidence 固定写“未提供”。
- severity（可选）：严重程度分级。  
  - 类比：口头警告、罚单、吊销驾照的区别。
  - 说明：`error` 级别不阻断输出，仅提示。

## warning 字段标准化模板（选定：模板 B）

- code：使用层级化命名。
  - 结构类：`STRUCTURE.MISSING_SECTION`
  - 内容类：`CONTENT.MISSING_METRIC`
  - 数值类：`NUMERIC.THRESHOLD_BREACH`
  - 合规类：`COMPLIANCE.PROHIBITED`
  - 语义类：`SEMANTIC.NO_BASIS`
- section：固定为 8 个章节名之一（必须严格一致）。
- reason：编号模板：`<问题类型>（<编号>）<原因>`。
- 编号规则：固定前缀 + 三位序号（例如：`STRUCTURE(001)`、`CONTENT(002)`）。
- 问题类型枚举：`结构缺失` / `内容缺失` / `数值超标` / `合规禁止` / `语义无依据`。
- 需要维护编号对照表（编号 <-> 含义）。
- 编号对照表位置：`finguard/docs/risk/warnings_index.md`。
- 对照表字段（选定 B）：`code` / `reason_template` / `section` / `basis_template` / `trigger_terms_rule`。
- 对照表格式：YAML 列表。
- 对照表顶部需包含元数据（title/tags/last_updated/source/status）。
- trigger_terms：命中词或数值数组，固定为数组（例如：`["负债率 65%"]`）；无触发词时使用空数组 `[]`。
- basis：固定格式 `阈值=<...>；来源=<...>`（非数值类写语义阈值，例如：`阈值=必须包含合规声明`）。
- evidence：原文片段字符串（必填）。
- 缺失类 warning 的 evidence 固定写“未提供”。
- severity：可选严重级别，规则：`COMPLIANCE.*` => `error`；`STRUCTURE.*` / `CONTENT.*` / `NUMERIC.*` / `SEMANTIC.*` => `warn`；`info` 预留；`error` 不阻断输出。

来源枚举（选定 A）：`模板要求` / `守拙价值多元化基金理念` / `业务规则`

## 关键结构与要求

### A. response 的边界
- response 是“公司基本面分析文本”。
- 不包含资产配置比例、仓位、买卖指令或收益承诺。

### B. 强制模板（标题固定，建议按顺序输出；顺序不触发 warning）
1) 公司信息
2) 财务状况（定量指标，含好价格判断）
3) 商业模式与护城河
4) 管理层结构与风格
5) 优点与风险点
6) 预期收益（模型估算，必须注明不确定性）
7) 结论（分析性总结）
8) 合规与数据声明

### C. 指标与阈值口径
- 负债率不得超过 50%。
- PEG 不得高于 2。
- ROE 必须大于 20%。
- 若文本出现数值，才做阈值判断。
- 若做“偏高/偏低”语义判断，必须写明阈值与理论来源。
- 财务指标未标明单位或口径时，不触发 warning（以简便为优先）。

### D. 预期收益公式（必写）

```text
Expected = (ROE * (1 + CAGR_5y) + 1) * (PE_long_term / PE_current) - 1
```

必须说明：假设值、区间、不确定性。
必须包含具体数值代入；缺失时输出 warning 并写“暂无数据，无法判断。”。

### E. 缺失指标处理
- 缺失任何关键指标时：必须输出 warning。
- 同时在正文中写出“暂无数据，无法判断。”。
 - 公司信息字段缺失（公司名称/交易所或代码/行业或赛道/核心产品）需 warning，并在正文标注“暂无数据，无法判断。”。
 - 结论章节必须包含 1-3 条要点，缺失或超出需 warning。

### F. 引用来源（可选）
- 引用来源放在“合规与数据声明”章节中，作为可选子项。
- 固定格式：`来源N：<标题>｜<年份>｜<链接>`；若无法固定则原样输出。
- 若缺失引用来源，仍需输出 warning。
- 缺失引用来源的 `reason` 使用 `CONTENT(###)缺少引用来源`。

## 关键流程 / 步骤

1. LLM 按固定模板生成分析文本。
2. check_response 逐段检查：
   - 标题是否齐全（顺序不触发 warning）。
   - 是否包含必需指标与合规声明。
   - 若出现数值，是否触发阈值规则。
3. 汇总 warning，输出到 SSE 的 warning 事件。

## 小结

- response 是基本面分析文本，不是资产配置建议。
- 模板标题必须严格固定，缺一即 warning。
- 指标缺失要同时提示“暂无数据/无法判断”。
- 数值阈值只在明确数值时触发；语义判断必须有阈值与来源。
