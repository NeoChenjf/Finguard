# 2026-01-27 学习总览：Web / Drogon / JSON / src 路线

## 标题与目标

读完这份文档，你能用通俗语言解释：HTTP/URL 与请求响应、Drogon 如何接路由、JSON 如何构造与解析、以及 `src/` 主链路如何从请求走到 LLM 再返回。

## 整体学习路线

1. Web 基础：HTTP / URL / 请求与响应。
2. MVC 与 Drogon：路由注册、启动服务、构造响应。
3. JSON：在 C++ 中构造与读取 JSON。
4. src 链路：`main -> http_server -> routes -> llm_client`。
5. SSE 流式输出：为什么要 `data: ...\n\n`。
6. 构建与依赖：CMake + vcpkg + 常用库（drogon/spdlog/nlohmann/json/gtest）。
7. 测试与验证：curl 测试、Mock 路由思路。

## 学习详情

### 1) Web / HTTP / URL 基础

- HTTP 像“点餐”，请求是点单，响应是上菜。
- URL = 地址 + 路径 + 参数，确定“去哪家店、点哪道菜”。
- 请求包含：方法（GET/POST）、路径、请求头、请求体（JSON）。

### 2) MVC 与 Drogon

- MVC：Model（数据）、View（展示）、Controller（控制）。
- Drogon 是 C++ Web 框架，负责开服务、接请求、路由分发、返回响应。

### 3) JSON 在 C++ 中怎么用

- JSON 是文本，收到后要解析成对象才能取字段。
- 常见操作：
  - 构造响应体：`Json::Value body; body["status"] = "ok";`
  - 读取请求体：`req->getJsonObject()` + `isMember()` + `asString()`。

### 4) src 主链路（从请求到响应）

- `main.cpp` 只负责启动服务。
- `http_server.cpp` 负责注册路由并启动 Drogon。
- `routes.cpp` 解析请求、鉴权、调用 LLM、输出 SSE。
- `llm_client.cpp` 负责访问外部模型 API，失败时可能走 curl fallback。

### 5) SSE 流式输出

- SSE 让输出“边生成边返回”，提升体感速度。
- 每条事件以 `data: ` 开头，空行分隔：`\n\n`。

### 6) 构建与依赖（CMake / vcpkg）

- CMake 负责“生成构建脚本”，构建工具负责真正编译。
- vcpkg 用于安装依赖并与 CMake 集成。
- 常见依赖：drogon / spdlog / nlohmann-json / gtest。

### 7) 测试与验证（curl / mock）

- 用 curl 直接测试 `/health`、`/api/v1/plan`、`/api/v1/chat/stream`。
- Mock 路由可用于隔离外部依赖，便于测试。

## 小结

- HTTP/URL 定义“请求怎么来、去哪”。
- Drogon 负责“接单、处理、回菜”。
- JSON 是通用数据格式，C++ 里需手动构造/解析。
- src 主链路决定请求如何进入 LLM 并以 SSE 输出。

## 自测问题

1. 请求与响应各包含哪些关键部分？
2. 为什么需要路由注册？
3. JSON 为什么不能自动变成变量？
4. SSE 为什么要 `data: ...` + 空行分隔？
5. `main -> http_server -> routes -> llm_client` 的顺序是做什么的？
6. CMake 与构建工具（Ninja/MSBuild）的职责区别是什么？
7. vcpkg 在项目里解决什么问题？

## 答疑和自测问题答案

1. 请求包含方法/路径/头/体；响应包含状态码/响应体。
2. 路由让 URL 映射到处理函数，决定“谁来处理这类请求”。
3. JSON 是文本，需要解析器才能变成可读对象。
4. SSE 协议要求每条消息以 `data:` 开头，用空行分隔事件。
5. 这是从入口到业务处理再到外部调用的主链路。
6. CMake 生成构建脚本；构建工具执行编译与链接。
7. vcpkg 统一安装依赖并与 CMake 集成。
