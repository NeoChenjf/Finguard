# 2026-01-28 学习总览：代理/降级与构建部署

## 标题与目标

读完这份文档，你能说清：代理/降级/HTTP 状态的核心概念，并能走完一遍 CMake 构建与部署的基本流程。

## 整体学习路线

1. 先理解“直连 vs 代理 vs 降级（curl fallback）”。
2. 再掌握“绝对 URL + Host 头”的必要性。
3. 区分 ReqResult / HTTP status / timeout。
4. 理解 SSE 的流式推送含义。
5. 走一遍 CMake 构建与部署流程。

## 学习详情

### 1) 直连、代理、降级（curl fallback）

- 直连：自己把请求送到 LLM。
- 代理：先走“中转站”，需要明确最终地址。
- 降级：主路失败就改走备用路线（curl）。

### 2) 绝对 URL + Host 头

- 走代理时必须提供“完整目的地地址”（绝对 URL）与 Host 头。
- 原因：代理只负责中转，需要知道最终要找的服务是谁。

### 3) ReqResult / HTTP status / timeout

- ReqResult：路上是否出事故（DNS/TLS/连接失败）。
- HTTP status：送达后对方给的回执（200/401/500）。
- timeout：等太久没回，判定超时。

### 4) SSE 流式推送

- SSE 是“边生成边推送”，提升用户体感。
- 连接保持更久，用于持续推送事件。

### 5) CMake 构建与部署（简化流程）

1. 配置（configure）：生成构建脚本。
2. 构建（build）：编译与链接输出可执行文件。
3. 运行与验证：检查日志与接口调用。

## 真实命令片段（节选）

```powershell
# 直连
curl.exe -s -X POST https://example.com/path -H "Content-Type: application/json" -d '{"prompt":"hi"}'

# 走代理
curl.exe -s -X POST https://example.com/path -H "Content-Type: application/json" -d '{"prompt":"hi"}' --proxy http://127.0.0.1:7890
```

```powershell
cmake -S d:\AI_Investment\finguard -B d:\AI_Investment\finguard\build
cmake --build d:\AI_Investment\finguard\build --config Debug
```

## 小结

- 代理像中转站，降级像备用路线。
- ReqResult 看“路上事故”，HTTP status 看“对方回执”。
- CMake 负责生成构建脚本，build 负责真正编译。

## 自测问题

1. 为什么走代理时必须写“绝对 URL + Host 头”？
2. curl fallback 的作用是什么？
3. ReqResult 与 HTTP status 有什么区别？
4. SSE 与普通响应最大的不同是什么？
5. CMake configure 与 build 各负责什么？

## 答疑和自测问题答案

1. 代理需要完整地址才能把请求转发到正确的目标服务。
2. 当主路径失败时，提供备用通路保证功能可用。
3. ReqResult 描述网络链路是否可达，HTTP status 描述服务端处理结果。
4. SSE 是分段推送，普通响应一次性返回。
5. configure 生成构建脚本，build 执行编译与链接。

- 关闭 `use_curl_fallback`，对比主路失败时的行为。
