# 学习路线：理解 Phase2 里这些网络与验收概念

目标
- 用生活类比弄懂：代理注入/降级、http_proxy 字段、Host/绝对 URL、curl fallback、proxy 透传、失败/降级路径、ReqResult/HTTP status/timeout。

学习路线（循序渐进）
1) 先理解“请求像寄快递”
   - 你要把包裹（请求）送到远方（LLM）。可以直送（直连）或找中转站（代理）。
2) 认识“直连失败就绕路”
   - 失败后退回到备用路线（curl fallback），这就是“降级”。
3) 了解“地址写法的差别”
   - 直连只写“门牌号”（path），走代理时要写“完整地址”（绝对 URL）+ 写清“收件人名”（Host 头）。
4) 学会读“回执与状态”
   - HTTP status 是快递签收结果（200/401/500），ReqResult 是运输途中是否出车祸（DNS/TLS/超时）。
5) 用一条最小命令做验证
   - 用 curl 走代理和直连各试一次，观察返回。

核心概念（生活类比）
- 代理注入：
  - 类比：你不自己送快递，而是把包裹交给快递公司中转站；“注入”就是在系统里指定必须走这个中转站。
- 降级：
  - 类比：高铁停运时改坐大巴。虽然慢，但能到。
- http_proxy 字段：
  - 类比：你写在纸条上的“指定中转站地址”。
- 连接代理 Host/Port + 绝对 URL + Host 头：
  - 类比：对中转站说“把这个包送到 北京市朝阳区某某路1号（绝对地址）”，同时在包裹上写“收件人名字（Host）”。
- curl fallback：
  - 类比：本地自己打电话（curl）去下单，绕过系统内部的快递线路。
- proxy 透传：
  - 类比：你把“指定中转站”这条纸条也交给了备用快递员（curl），确保备用路线也走同一个中转站。
- 失败/降级路径：
  - 类比：主路堵车 → 走备用路 → 如果备用也堵，记录问题并给出临时结果。
- ReqResult / HTTP status / timeout：
  - ReqResult：路上出没出事故（DNS 找不到路/握手失败/连接断了）。
  - HTTP status：送达后对方给的回执（200 成功/401 无权限/500 服务器错）。
  - timeout：等太久没回信，判定超时。

关键流程/步骤（带最小示例）
1) 直连（不走代理）
```powershell
curl.exe -s -X POST https://example.com/path -H "Content-Type: application/json" -d '{"prompt":"hi"}'
```
2) 走代理（http_proxy）
```powershell
curl.exe -s -X POST https://example.com/path -H "Content-Type: application/json" -d '{"prompt":"hi"}' --proxy http://127.0.0.1:7890
```
3) 理解返回
- 若返回 200：说明服务端收到了。
- 若 ReqResult 失败：说明连路都没通（DNS/网络/TLS）。
- 若超时：说明等不到回信。

小结
- 直连=自己送；代理=中转站；降级=改走备用路线。
- HTTP status 是“对方签收的结果”，ReqResult 是“路上是否出事”。
