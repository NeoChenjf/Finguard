# 技术栈选择入门（Phase 7）

> 目标：看完后你能用白话解释 Vite、React、TypeScript、TailwindCSS、Lucide React、React Router 各自做什么，以及它们如何配合。

## 1) 学习路线（从易到难）

1. 先分清角色：谁是“开发工具”、谁是“页面框架”、谁是“语言增强”、谁是“样式方案”。
2. 再看协作关系：从“写代码”到“页面显示”完整走一遍。
3. 最后看一段最小可运行代码，把 6 个概念串起来。

## 2) 核心概念解释（白话 + 类比）

### Vite 是什么？
- 白话：Vite 是前端项目的“启动器 + 打包器”，负责本地开发启动和生产打包。
- 你可以理解成：它让你在开发时“秒开页面、改完立刻生效”。

### React 是什么？
- 白话：React 是用来搭页面的框架，把页面拆成很多可复用“小组件”。
- 你可以理解成：像搭积木，按钮、表单、卡片都可以复用。

### TypeScript 是什么？
- 白话：TypeScript 是 JavaScript 的“带类型版本”，能提前发现很多低级错误。
- 你可以理解成：写代码时多了“语法体检”，减少运行后才报错。

### TailwindCSS 是什么？
- 白话：TailwindCSS 是一套“现成样式类名”，你直接在标签上拼类名就能做 UI。
- 你可以理解成：像搭配衣服，不用自己从零裁布（写大量 CSS）。

### Lucide React 是什么？
- 白话：Lucide React 是 React 图标库，提供很多可直接用的 SVG 图标组件。
- 你可以理解成：UI 的“图标零件箱”，比如趋势箭头、设置齿轮。

### React Router 是什么？
- 白话：React Router 是页面路由管理工具，负责“URL 对应哪个页面组件”。
- 你可以理解成：商场导览图，`/settings` 去设置页，`/chat` 去问答页。

### 一个生活类比（盖房子）
- Vite：施工队和工具车（开工快、效率高）。
- React：房屋结构（客厅、卧室这些模块化空间）。
- TypeScript：施工图纸标注（尺寸和规则更明确，少返工）。
- TailwindCSS：装修材料库（现成墙面/配色/间距方案）。
- Lucide React：门牌和指示图标。
- React Router：楼层导览和门牌号系统。

## 3) 关键流程（可复刻）

1. 用 Vite 创建 React + TypeScript 项目。
2. 安装 TailwindCSS，开始用类名写样式。
3. 安装 React Router，配置 `/`、`/settings`、`/chat` 等路由。
4. 安装 Lucide React，在按钮或标题中加入图标。
5. 运行开发服务，边改边看页面效果。

## 4) 真实代码片段（逐行注释）

```tsx
import { BrowserRouter, Routes, Route, Link } from "react-router-dom";
import { TrendingUp } from "lucide-react";

function HomePage() {
  return (
    <div className="min-h-screen bg-slate-950 text-white p-6">
      <h1 className="text-2xl font-bold flex items-center gap-2">
        <TrendingUp className="w-6 h-6 text-emerald-400" />
        FinGuard 首页
      </h1>
      <Link to="/settings" className="inline-block mt-4 underline">
        去设置页
      </Link>
    </div>
  );
}

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<HomePage />} />
      </Routes>
    </BrowserRouter>
  );
}
```

- 第 1 行：从 `react-router-dom` 引入路由相关组件（Router、路由集合、单条路由、链接）。
- 第 2 行：从 `lucide-react` 引入趋势图标组件。
- 第 4 行：定义首页组件 `HomePage`。
- 第 5 行：组件返回 JSX（页面结构）。
- 第 6 行：最外层容器用 TailwindCSS 控制高度、背景、文字颜色、间距。
- 第 7 行：标题区域用 TailwindCSS 控制字号、加粗、横向排列。
- 第 8 行：渲染 Lucide 图标，并用 TailwindCSS 指定大小与颜色。
- 第 9 行：标题文字内容。
- 第 11-13 行：`Link` 组件跳转到 `/settings`，并设置基础样式。
- 第 18 行：定义根组件 `App`。
- 第 20 行：`BrowserRouter` 开启浏览器路由模式。
- 第 21-23 行：配置路由规则：访问 `/` 时渲染 `HomePage`。

## 5) 小结 + 自测题（含答案）

### 小结
- 这 6 个概念不是重复功能，而是“分工合作”：Vite 管工程效率，React 管页面组件，TypeScript 管代码安全，TailwindCSS 管样式效率，Lucide 管图标，React Router 管页面跳转。

### 自测题
1. 哪个工具负责“URL 切换到不同页面”？  
   答：React Router。
2. 哪个工具让你“改代码几乎立即看到页面变化”？  
   答：Vite。
3. 哪个工具让你少写传统 CSS 文件、直接在标签上写样式？  
   答：TailwindCSS。
