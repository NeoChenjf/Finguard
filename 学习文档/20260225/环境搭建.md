# 环境搭建入门（asyncio / aiohttp / pip / Mock LLM / ThreadingHTTPServer）

## 标题与目标

目标：你能理解并上手这 4 件事：`asyncio`、`aiohttp`、`pip install`、`Mock LLM server（虚拟下游）`，并知道 `ThreadingHTTPServer` 如何做“可配端口”。

## 学习路线/分层

1. 先搞清术语：`asyncio` 和 `aiohttp` 分别做什么。
2. 再搞清安装：`pip install` 到底在做什么。
3. 再搞清架构：Mock LLM server 和“虚拟下游”在系统里的位置。
4. 最后搞清实现：`ThreadingHTTPServer` 如何支持 `--port` 改端口。

## 核心概念解释（带生活类比）

- `asyncio`：Python 的异步调度框架。像一个前台同时盯很多桌，不会卡死在一桌。
- `aiohttp`：基于 `asyncio` 的 HTTP 客户端/服务端库。像“会异步点单和收单”的网络工具。
- `pip install`：Python 包管理安装命令。像“给厨房补原料”，把代码依赖装到当前环境。
- Mock LLM server：假的 LLM 下游服务。像“训练用假厨房”，可控、稳定、可重复。
- 虚拟下游：你服务真实会调用的外部依赖（这里是 LLM API）的替身。目的不是“聪明回答”，而是“稳定测试”。
- `ThreadingHTTPServer`：每个请求由独立线程处理的 HTTP 服务器，适合做简单并发 Mock。
- 可配端口：启动时传 `--port` 决定监听端口，不把端口写死在代码里。
- `18080` / `28080`：都是端口号（服务入口编号）。`18080` 常作为默认示例，`28080` 常作为冲突时的备用示例。
- 为什么这样写：端口可能被其他程序占用，写成 `--port` 才能快速切换，避免 `Address already in use`。

## 关键流程/步骤（可复刻）

1. 创建并激活虚拟环境（推荐）
2. `pip install` 安装 `aiohttp`
3. 启动项目内 Mock LLM server（虚拟下游）
4. 用异步客户端发请求验证
5. 需要时切换端口（例如 18080 改 28080）

## 真实命令/代码示例（带逐行注释）

### 1) `pip install` 最小实操

```powershell
# 进入项目根目录
Set-Location d:\AI_Investment
# 创建虚拟环境（首次）
python -m venv .venv
# 激活虚拟环境（PowerShell）
.\.venv\Scripts\Activate.ps1
# 升级 pip，避免老版本兼容问题
python -m pip install --upgrade pip
# 安装 aiohttp（异步 HTTP 库）
pip install aiohttp
```

说明：
- `pip install` 会把包安装到当前激活的 Python 环境。
- 你如果不激活 `.venv`，包可能会被装到系统 Python，后续容易混乱。

### 2) 使用项目里的 Mock LLM server（虚拟下游）

```powershell
# 在项目根目录启动 mock 下游，端口 18080
python tools/mock_llm_server.py --host 127.0.0.1 --port 18080
```

说明：
- 这个脚本实现了 `/chat/completions` 和 `/health`。
- 它支持通过 prompt 里的指令模拟延迟/错误（例如 `__mock:status=500`、`__mock:delay_ms=2000`）。
- 这就是“虚拟下游”：可控地模拟真实 LLM，方便压测和故障注入。
- `127.0.0.1:18080` 的含义是“本机地址 + 18080 端口”。
- 如果提示端口占用，就改成 `--port 28080`，客户端 URL 也要同步改端口。

### 3) `asyncio + aiohttp` 异步请求最小示例

```python
import asyncio  # 异步调度器
import aiohttp  # 异步 HTTP 客户端

URL = "http://127.0.0.1:18080/chat/completions"  # 指向 mock 下游

async def one_call(session: aiohttp.ClientSession) -> None:
    payload = {"messages": [{"role": "user", "content": "hello"}]}  # 构造请求体
    async with session.post(URL, json=payload) as resp:  # 发送异步 POST
        text = await resp.text()  # 异步读取响应文本
        print(resp.status, text[:120])  # 打印状态码和前 120 字符

async def main() -> None:
    async with aiohttp.ClientSession() as session:  # 复用连接
        tasks = [asyncio.create_task(one_call(session)) for _ in range(5)]  # 并发 5 个请求
        await asyncio.gather(*tasks)  # 等待全部完成

if __name__ == "__main__":
    asyncio.run(main())  # 启动事件循环
```

说明：
- `asyncio` 负责并发调度。
- `aiohttp` 负责异步发 HTTP 请求。
- 这套模型比“多开很多进程/任务”更轻量，适合压测脚本。

### 4) `ThreadingHTTPServer` 可配端口最小骨架

```python
import argparse  # 解析命令行参数
from http.server import BaseHTTPRequestHandler, ThreadingHTTPServer  # 标准库 HTTP 服务器

class Handler(BaseHTTPRequestHandler):
    def do_GET(self):  # 处理 GET 请求
        if self.path == "/health":  # 健康检查端点
            self.send_response(200)  # 返回 200
            self.end_headers()  # 结束响应头
            self.wfile.write(b"ok")  # 返回响应体
            return
        self.send_response(404)  # 非法路径返回 404
        self.end_headers()  # 结束响应头

def main():
    ap = argparse.ArgumentParser()  # 创建参数解析器
    ap.add_argument("--host", default="127.0.0.1")  # 可配 host
    ap.add_argument("--port", type=int, default=18080)  # 可配 port
    args = ap.parse_args()  # 读取命令行参数

    server = ThreadingHTTPServer((args.host, args.port), Handler)  # 绑定 host:port
    print(f"listening on {args.host}:{args.port}")  # 输出监听地址
    server.serve_forever()  # 持续提供服务

if __name__ == "__main__":
    main()  # 启动入口
```

说明：
- `--port` 就是“可配端口”的关键。
- 启动命令改成 `python xxx.py --port 28080`，端口就切到 `28080`。
- `18080` 和 `28080` 没有固定业务语义，它们只是可替换的高位端口示例（通常不需要管理员权限）。

## 小结 + 自测问题（含答案）

小结：
- `asyncio` 是异步调度器，`aiohttp` 是基于它的网络库。
- `pip install` 是把依赖装进当前 Python 环境。
- Mock LLM server 是“虚拟下游”，用来稳定复现下游行为（成功、慢、报错）。
- `ThreadingHTTPServer + argparse --port` 可以实现端口可配置。

自测题：
1. `asyncio` 和 `aiohttp` 的分工分别是什么？
2. 为什么建议先激活 `.venv` 再 `pip install`？
3. “虚拟下游”在压测里最核心的价值是什么？
4. 如何把 mock server 从 18080 改到 28080？
5. 为什么不建议把端口写死成一个值？

答案：
1. `asyncio` 负责并发调度，`aiohttp` 负责异步 HTTP 通信。
2. 避免包装到系统 Python，保证依赖隔离、环境可复现。
3. 可控、稳定、可重复，能精确模拟延迟和错误。
4. 启动时传 `--port 28080`。
5. 因为端口可能被占用；写成可配置参数能在不同机器和场景下快速切换。
