# 连接池与单例复用入门

## 标题与目标

目标：理解“连接池”和“单例复用”分别是什么、有什么区别、什么时候该用哪一种，并能在项目里落地。

## 学习路线/分层

1. 先理解：为什么每请求新建客户端会慢。
2. 再区分：连接池 vs 单例复用。
3. 再看落地：代码怎么写、参数怎么配。
4. 最后避坑：线程安全、生命周期、资源上限。

## 核心概念解释（带生活类比）

- 连接池：预先准备一批可复用连接，请求来时借用，用完归还。像餐厅门口有一排随时可用外卖车。
- 单例复用：整个进程只创建一个客户端实例，大家共用。像全店只用一个调度台。
- 每请求新建客户端：每次都重新建连接、握手、可能重新 DNS。像每接一单都重新买一辆车。

关键区别：
- 单例复用是“对象层面复用一个客户端实例”。
- 连接池是“连接层面复用多条连接”。
- 常见实现是：一个单例客户端内部维护连接池。

## 关键流程/步骤（可复刻）

1. 确认热点路径
   - 看代码是否每请求都 `new HttpClient`。

2. 改成复用模型
   - 第一步：先改为单例客户端，避免重复创建对象。
   - 第二步：启用或调优连接池，避免重复建连。

3. 设置基础参数
   - 连接池大小（最大连接数）
   - 空闲超时
   - 请求超时

4. 验证效果
   - 对比 P95/P99、RPS、错误率、RSS。

## 真实代码片段或命令示例（带逐行注释）

### 1) 反例：每请求新建客户端（伪代码）

```cpp
void handle_request() {
    auto client = HttpClient::newHttpClient(host_url); // 每次请求都创建客户端对象
    auto resp = client->send(req);                     // 可能触发新建连接与额外开销
}
```

说明：
- 功能能跑。
- 高并发下容易造成延迟抖动和资源浪费。

### 2) 正例：单例复用客户端（伪代码）

```cpp
class ClientHolder {
public:
    static HttpClientPtr get() {
        static HttpClientPtr client = HttpClient::newHttpClient(host_url); // 只初始化一次
        return client;                                                      // 后续请求复用
    }
};

void handle_request() {
    auto client = ClientHolder::get(); // 复用同一个客户端实例
    auto resp = client->send(req);     // 避免重复创建对象
}
```

说明：
- 先把“对象重复创建”问题去掉。
- 再看该客户端是否支持连接池参数调优。

### 3) Python aiohttp 的连接池示例（真实可运行）

```python
import asyncio  # 异步调度
import aiohttp  # 异步 HTTP 客户端

URL = "http://127.0.0.1:18080/chat/completions"  # 下游地址

async def one_call(session: aiohttp.ClientSession) -> None:
    payload = {"messages": [{"role": "user", "content": "hello"}]}  # 请求体
    async with session.post(URL, json=payload) as resp:  # 复用同一 Session 内连接池
        _ = await resp.text()  # 读取响应，确保连接可回收复用

async def main() -> None:
    connector = aiohttp.TCPConnector(limit=100)  # 连接池上限 100
    timeout = aiohttp.ClientTimeout(total=10)    # 总超时 10 秒
    async with aiohttp.ClientSession(connector=connector, timeout=timeout) as session:
        tasks = [asyncio.create_task(one_call(session)) for _ in range(50)]  # 并发 50
        await asyncio.gather(*tasks)  # 等全部完成

if __name__ == "__main__":
    asyncio.run(main())
```

说明：
- `ClientSession` 是“单例复用对象”。
- `TCPConnector(limit=100)` 是“连接池容量”。

## 小结 + 自测问题（含答案）

小结：
- 单例复用解决“重复创建客户端对象”。
- 连接池解决“重复创建网络连接”。
- 两者不是二选一，通常是组合使用。
- 性能优化要用指标验证，不能靠感觉。

自测题：
1. 单例复用和连接池的本质区别是什么？
2. 为什么“每请求 new 客户端”在高并发会变慢？
3. 连接池太小和太大分别会有什么问题？
4. 为什么要在优化后再看 P95/P99 和 RSS？

答案：
1. 单例复用是对象复用，连接池是连接复用。
2. 会重复触发对象创建、建连、握手等开销。
3. 太小会排队等待，太大会占用过多资源并增加管理成本。
4. 因为优化是否有效必须由指标证明。
