# 学习文档目录说明（给 agent 与用户）

目的
- 该目录用于沉淀“学习资料、学习路线与阶段性记录”。
- 以日期文件夹归档，便于回溯学习过程与上下文。

目录结构约定
- `学习文档/20260127/`：历史资料归档（原始学习文档迁入）。
- `学习文档/20260128/`：当天学习与讲解内容（新增学习路线、笔记、问题清单）。

维护协作规范
- 新增学习内容时：
  - 放入当天日期文件夹（`YYYYMMDD`）。
  - 文件名用中文直观命名（如“学习路线.md”“概念解释.md”）。
- 修改旧资料时：
  - 保持原文件内容不被大幅重写，新增内容可追加到文件末尾。
  - 必要时在文件末尾加一行“更新记录：YYYY-MM-DD …”。
- 不在学习文档中放真实密钥或敏感信息。

与用户协作方式
- 用户提出不懂概念时：先写“概念解释 + 学习路线”，再配最小可复制示例。
- 用户要求整理时：先确认归档日期与文件位置，再移动或新增文件。
- 若需要新增规则或模板：先在当天文件夹建新文件，确认后再扩展到公共模板。
