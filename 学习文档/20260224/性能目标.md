# 性能目标入门（P95/P99、RPS、退化）

## 标题与目标

目标：用小白能懂的方式解释 P95、P99、RPS、以及“混合场景 P95 退化”的含义，能读懂 Phase5 的性能目标表。

## 学习路线/分层

1) 先理解“百分位”是什么
2) 再理解 P95 / P99 在性能里代表什么
3) 再理解 RPS 是什么
4) 最后解释“混合场景 P95 退化”

## 核心概念解释（带生活类比）

- 百分位：把所有响应时间从快到慢排队，P95 就是“前 95% 都不比它慢”的那个点。
- P95：像“95 个客人中，最慢的那 5 个被排除后”的边界时间。
- P99：像“99 个客人中，最慢的那 1 个被排除后”的边界时间，更关注极端慢请求。
- RPS（Requests Per Second）：每秒能处理多少请求，像“每秒能出几道菜”。
- 退化：同样的接口、同样的压力，但混合场景下反而变慢或变差，就是“退化”。

## 关键流程/步骤（可复刻）

1) 收集响应时间列表
2) 排序并找出 P95 / P99
3) 统计每秒完成请求数（RPS）
4) 比较单场景与混合场景的 P95 差距，计算退化比例

## 真实命令/代码示例（带逐行注释）

### 1) 计算“退化比例”的伪代码示例

```text
# 单场景 P95 = 200ms
# 混合场景 P95 = 260ms
# 退化比例 = (260 - 200) / 200 = 30%
```

说明：
- 若退化 < 30%，说明混合场景没有“明显变慢”。

### 2) RPS 概念示例

```text
# 10 秒内完成 500 个请求
# RPS = 500 / 10 = 50
```

## 小结 + 自测问题（含答案）

小结：
- P95 / P99 是“慢请求的边界指标”。
- P99 更关注“最慢那 1%”。
- RPS 是每秒处理请求数。
- 退化是“混合场景比单场景更慢”的程度。

自测题：
1) P95 和 P99 的区别是什么？
2) RPS 是什么？
3) 单场景 P95=200ms，混合场景 P95=260ms，退化是多少？

答案：
1) P95 看前 95% 的边界，P99 看前 99% 的边界，P99 更关注极端慢请求。
2) 每秒处理的请求数。
3) (260-200)/200 = 30%。

## 下一步建议

1) 结合实际压测结果算出 P95/P99。
2) 标记混合场景的退化比例。
3) 对比优化前后 RPS 的变化。
