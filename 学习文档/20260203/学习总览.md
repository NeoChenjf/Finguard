# 2026-02-03 学习总览：档案/规则/路由/规则配置

## 标题与目标

读完这份文档，你能按“学习路线”理解四块内容，并能说清：档案存取、规则加载与检查、路由接入流程、以及 rules.yaml 哪些字段真实生效。

## 整体学习路线

## 学习详情

### A. profile_store（档案存取）

**核心概念（含类比）**

- 档案文件路径：`<当前工作目录>/config/profiles.json`。
  - 类比：账本固定放在一个抽屉里，只有知道抽屉位置才能找到。
- 根节点（root）：`profiles.json` 顶层 JSON 对象。
- version：档案结构版本号，便于未来升级兼容。
- 线程安全：互斥锁保证一次只有一个线程读写文件。
  - 类比：公共厕所的锁，同一时间只允许一个人进入。

**关键流程 / 步骤**

1. `profile_path()` 拼出档案路径。
2. `load_profiles()`：文件不存在就创建默认结构；存在则解析并补齐字段。
3. `save_profiles()`：创建目录后写入 JSON。
4. `upsert_profile()` / `load_profile()`：对外读写入口。

**真实代码片段**

```cpp
std::filesystem::path profile_path() {
    return std::filesystem::current_path() / "config" / "profiles.json";
}
```

```cpp
root["users"][user_id]["questionnaire"] = questionnaire;
```

**默认结构示例**

```json
{
  "version": 1,
  "users": {}
}
```

### B. rule_engine（规则引擎）

**核心概念（含类比）**

- 规则配置文件：`<当前工作目录>/config/rules.yaml`。
  - 类比：公司手册，不在时就按通用守则。
- 默认值兜底：`load_defaults()` 先写入基础规则。
- 请求检查：`check_request()` 根据问卷 + 关键词输出 warning。

**关键流程 / 步骤**

1. `load_config()` 先写默认值，再读 `rules.yaml`。
2. `parse_yaml()` 只解析“硬编码路径”的字段。
3. `check_request()` 读取问卷字段，生成 warning。

**真实代码片段**

```cpp
const auto path = std::filesystem::current_path() / "config" / "rules.yaml";
```

```cpp
if (questionnaire.isMember("investor_profile") && questionnaire["investor_profile"].isString()) {
    investor_profile = questionnaire["investor_profile"].asString();
} else {
    result.warnings.push_back("missing_investor_profile_defaulted");
}
```

**路径易错点与解决**

- `rules.yaml` 路径依赖当前工作目录。
- 解决方式：固定工作目录 / 使用绝对路径 / 基于可执行文件定位。

### C. routes.cpp（路由接入与 SSE）

**核心概念（含类比）**

- 路由注册：把 URL 路径绑定到处理函数。
  - 类比：把来访问题分配到对应窗口。
- SSE（流式输出）：边生成边推送，让用户实时看到内容增长。

**关键流程 / 步骤**

`/api/v1/profile/upsert`
1. 读 `X-User-Id`，缺失返回 400。
2. 校验 `questionnaire` 为对象。
3. 调用 `risk::upsert_profile` 写档案。

`/api/v1/chat/stream`
1. 读 `prompt`。
2. 读取档案（可能失败）。
3. 加载规则引擎并执行 `check_request()`。
4. 校验 `X-API-Key`。
5. 调用 `stream_chat()` 并组装 SSE 事件流。

**SSE 事件顺序**

- token → cite → metric → warning → done

**真实代码片段**

```cpp
app().registerHandler("/api/v1/profile/upsert",
    [](const HttpRequestPtr &req, std::function<void(const HttpResponsePtr &)> &&cb) {
        const auto user_id = req ? req->getHeader("X-User-Id") : "";
        if (user_id.empty()) {
            Json::Value body;
            body["error"] = "missing_user_id";
            auto resp = HttpResponse::newHttpJsonResponse(body);
            resp->setStatusCode(k400BadRequest);
            cb(resp);
            return;
        }
        // ...
    },
    {Post});
```

### D. rules.yaml（字段与生效方式）

**会生效的字段（被解析）**

- `allocation.gold_percent`
- `allocation.bonds.default_age`
- `risk_limits.min_single_asset_percent`
- `warnings.message_template`
- `warnings.keyword_rules.forbidden_keywords`
- `allocation.equities.individual_stock.profile_limits.<profile>.<field>`

**不会生效的字段（目前不解析）**

- 顶层：`version`、`name`
- `inputs.*`
- `allocation.bonds.method/etf/examples`
- `allocation.equities.index_allocation.*`
- `allocation.equities.individual_stock.profile_limits.*.requirements`
- `stock_selection.*`
- `warnings.include_trigger_terms/include_reason`

**如何验证**

- 改动“会生效字段”，再调用 `/api/v1/chat/stream` 看 warning 是否变化。
- 改动“不会生效字段”，一般不会有变化。

## 小结

- 档案、规则、路由、配置四块内容是“读取档案 → 执行规则 → 生成 SSE 输出”的完整链路。
- 只有被 `parse_yaml()` 明确解析的字段才会真正生效。

## 自测问题

1. `profiles.json` 会存在哪个目录？
2. `profiles.json` 不存在时默认结构是什么？
3. 为什么 `upsert_profile` 与 `load_profile` 要加互斥锁？
4. `load_config()` 为什么先调用 `load_defaults()`？
5. `parse_yaml()` 解析哪些路径？
6. `check_request()` 在哪些情况下会产生 warning？
7. `/api/v1/profile/upsert` 缺少 `X-User-Id` 会返回什么？
8. `/api/v1/chat/stream` 为什么校验 `X-API-Key`？
9. 为什么要设置 `text/event-stream` 与 `keep-alive`？
10. `rules.yaml` 里哪些字段只是“说明”，不会生效？

## 答疑和自测问题答案

1. 在当前工作目录下的 `config/profiles.json`。只有触发写入时才会真正创建文件。
2. 默认结构是：`{ "version": 1, "users": {} }`。
3. 防止并发读写同一文件造成竞态（写乱、读到半成品）。
4. 为了兜底：即使读取配置失败也有基础规则可用。
5. 只解析硬编码路径（见“会生效字段”列表）。
6. 问卷字段缺失、关键词命中、画像限制冲突或比例过小都会产生 warning。
7. 返回 400 + `missing_user_id`。
8. 用于接口访问控制；比对的是服务端配置的 `api_key`，不一定与阿里云绑定。
9. `text/event-stream` 是 SSE 专用类型，`keep-alive` 保持连接不断开，便于持续推送。
10. 例如 `inputs.*`、`stock_selection.*` 等，目前只是说明性字段。
