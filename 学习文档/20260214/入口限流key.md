# 入口限流 key：route 与 query

目标：用最简单的话说明 `route` 和 `query` 是什么，并知道它们在限流里怎么用。

## 学习路线（从易到难）
1. 先区分 URL 的路径与查询参数。
2. 再理解限流 key 为什么常用 route。
3. 最后理解不包含 query 的原因。

## 核心概念解释（生活类比）
- 把 URL 当作“送餐地址”。
- `route` 是小区楼栋号。
- `query` 是“不要辣、少盐”的备注。

## 关键流程（可复刻）
1. 找到 `?` 之前的部分，作为 `route`。
2. 找到 `?` 之后的部分，作为 `query`。
3. 入口限流 key 只用 `route`，不要把 `query` 拼进去。
4. URL 里 `?`是分隔符，用来把“路径”和“查询参数”分开。

## 真实示例（逐行注释）
```text
/api/v1/plan?page=1&sort=asc
```
- `/api/v1/plan` 是 `route`。
- `page=1&sort=asc` 是 `query`。
- `page=1` 表示第 1 页 `sort=asc` 表示按升序排序

## 小结
- `route` 是路径，`query` 是查询参数。
- 限流 key 通常只用 `route`，避免被参数打散。

## 自测题（含答案）
1) `route` 是 URL 的哪一部分？
- 答案：`?` 之前的路径部分。

2) 为什么限流 key 通常不包含 `query`？
- 答案：`query` 会把请求拆成很多小组，限流不容易触发。

## 下一步建议
1. 从项目里的日志或接口文档中找 2 个 URL，标出 route 与 query。
2. 想一想：如果把 query 加入 key，会出现什么问题。
