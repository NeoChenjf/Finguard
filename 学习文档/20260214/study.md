# Phase 4 术语教学：限流、熔断、超时、可观测

目标：用小白能懂的方式解释常见名词，并能读懂配置里的关键字段。

## 学习路线（从易到难）
1. 先理解“限流”和“429 错误体”是怎么保护入口的。
2. 再理解“熔断器”的失败口径、窗口/阈值、半开与恢复。
3. 最后理解“退避、指标开关与输出、采样/标签”的作用。

## 核心概念解释（生活类比）
- 限流像“店门口发号”：一次只能放多少人进店，防止店里挤爆。
- 熔断像“电路保险丝”：线路故障太多就先断开，过一会儿再试着恢复。
- 退避像“打电话占线后隔一会再拨”：避免一直连打导致更堵。
- 指标与日志像“运营看板”：老板看总数据，店长看每单细节。

## 关键名词逐个讲清楚
1) key 维度
- 白话：限流时用什么“标识”来分组。比如按用户、按路由、按 IP。
- 作用：决定“谁和谁共用一个桶”。粒度太粗会误伤，太细会失效。

2) 429 错误体
- 白话：被限流时返回的标准错误说明。
- 作用：让客户端知道“被限流了、多久再试”。

3) 熔断器策略
- 白话：当下游不稳定时，先短路保护，再试探恢复。
- 由四个部分组成：失败口径、窗口/阈值、半开探测、恢复条件。

4) 失败口径
- 白话：哪些错误算“失败”。常见是 5xx、超时、网络错误。
- 作用：决定熔断统计什么。

5) 窗口/阈值
- 白话：在多长时间内（窗口）失败率超过多少（阈值）就熔断。
- 作用：避免偶发错误触发熔断，也避免长期错误不处理。

6) 半开探测
- 白话：熔断后先放少量请求试试，看是否恢复。
- 作用：避免一直断开导致服务永远不恢复。

7) 恢复条件
- 白话：半开探测成功多少次就恢复为正常。
- 作用：给恢复一个明确门槛。

8) 退避（Backoff）
- 白话：重试前先等一会儿再试。
- 作用：减少雪崩，让下游有喘息。

9) 指标开关与输出
- 白话：是否启用指标，以及指标从哪里输出（例如 /metrics）。
- 作用：让监控系统能抓到 P95/P99、熔断次数等。

10) 采样或标签
- 白话：采样=只记录一部分请求；标签=给指标加维度（如 route、status）。
- 作用：采样降低成本，标签让数据更好分析。

## 关键流程（可复刻）
1. 先确定限流 key 维度（例如 user_id + route）。
2. 定义 429 错误体，客户端能识别 retry_after。
3. 定义熔断失败口径、窗口/阈值与半开探测。
4. 配置重试与退避，避免连续打爆下游。
5. 开启指标输出与必要标签，方便观测。

## 真实配置示例（逐行注释）
```yaml
rate_limit: # 限流配置根节点
  entry: # 入口限流
    key: "user_id + route" # key 维度
    limit_per_sec: 10 # 每秒允许通过的速率
    burst: 20 # 桶容量，允许短暂突发
    on_reject_status: 429 # 被限流时的状态码
    on_reject_body: "{\"error\":{\"code\":\"RATE_LIMITED\",\"message\":\"rate limited\",\"retry_after_ms\":1000}}" # 429 错误体
circuit_breaker: # 熔断配置根节点
  failure_types: ["5xx", "timeout", "network"] # 失败口径
  window_sec: 60 # 统计窗口（秒）
  failure_rate: 0.5 # 失败率阈值（50%）
  half_open_max_requests: 2 # 半开探测允许的请求数
  recovery_successes: 3 # 连续成功次数达到则恢复
retry: # 重试与退避
  max_attempts: 1 # 最大重试次数
  backoff_ms: [200, 400] # 退避时间（毫秒）
observability: # 可观测
  metrics_enabled: true # 指标开关
  metrics_path: "/metrics" # 指标输出地址
  sampling_rate: 0.1 # 采样比例（10%）
  labels: ["route", "status"] # 指标标签
```

## 小结
- key 维度决定限流粒度，429 错误体统一客户端处理。
- 熔断器关注失败口径、窗口/阈值、半开探测与恢复条件。
- 退避、指标开关、采样与标签决定系统的稳定性与可观测成本。

## 自测题（含答案）
1) 为什么限流 key 太细会导致限流失效？
- 答案：过细会把请求拆成很多小组，每组都不够“多”触发限流。

2) 熔断的失败口径为什么通常不包含 4xx？
- 答案：4xx 多为客户端问题，不代表下游不可用。

3) 半开探测的目的是什么？
- 答案：在断路后少量试探恢复，避免永久断开。

4) 退避的作用是什么？
- 答案：给下游喘息时间，降低重试风暴。
