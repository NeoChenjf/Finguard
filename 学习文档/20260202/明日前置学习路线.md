## 核心概念解释
### 通俗版（生活类比）
- **档案存储**像“会员卡信息库”，先登记再服务。
- **规则引擎**像“门店店长”，负责把关规则与风险提示。
- **路由接入**像“把店长安排到前台”，确保每次服务都能触发检查。

### 专业版（准确解释）
- **profile_store**：以 `profiles.json` 存取用户问卷档案，支持 upsert 与 load。
- **rule_engine**：读取 `rules.yaml`，在请求侧做关键词与画像规则判断。
- **routes.cpp**：在 `/api/v1/chat/stream` 中串联“读取档案 → 规则检查 → LLM 调用 → SSE 输出”。

## 关键流程 / 步骤
1. 阅读 `finguard/src/risk/profile_store.*`：理解档案文件路径与存取逻辑。
2. 阅读 `finguard/src/risk/rule_engine.*`：关注 `load_config` 与 `check_request`。
3. 阅读 `finguard/src/server/routes.cpp`：找 `/api/v1/profile/upsert` 与 `/api/v1/chat/stream` 的接入逻辑。
4. 对照 `finguard/config/rules.yaml`：理解字段与规则实际生效方式。
