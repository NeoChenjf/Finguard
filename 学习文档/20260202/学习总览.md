# 2026-02-02 学习总览：YAML 与整体调用流程

## 标题与目标

读完这份文档，你能读懂 `rules.yaml` 的结构、理解问卷与档案的差异，并能描述“main → routes → LLM → SSE”的整体流程。

## 整体学习路线

1. YAML 基础：缩进、键值、列表规则。
2. rules.yaml：理解字段层级与修改方式。
3. 问卷提交方式：内联提交 vs 档案持久化。
4. LLM 上下文：档案在后端不等于 LLM 自动知道。
5. 整体调用链路：从 main 到 SSE 输出。

## 学习详情

### 1) YAML 基础

- YAML 是人类可读的配置格式。
- 缩进决定层级；`-` 表示列表项。
- 机器通过解析器把 YAML 转成树状结构。

### 2) rules.yaml 的读法

- 先找顶层键（如 allocation / risk_limits / warnings）。
- 修改字段时只改值，不改缩进结构。

### 3) 问卷提交方式

- **内联提交**：每次请求都带 `questionnaire`。
- **档案持久化**：先写 `/api/v1/profile/upsert`，聊天时按 `X-User-Id` 读取。

### 4) 档案与 LLM 上下文

- 档案只在后端有效，LLM 不会自动知道。
- 若需要个性化回答：需把档案摘要注入 prompt/system。

### 5) 从 main 到 SSE 的整体流程

- `main` 启动服务。
- `http_server` 注册路由并监听端口。
- `routes` 解析请求、鉴权、调用 LLM。
- 结果以 SSE 事件（token/cite/metric/warning/done）返回。

## 小结

- YAML 通过缩进表达层级，是规则配置的基础格式。
- 问卷可以内联或持久化，项目里使用档案方式。
- 整体链路决定了请求如何进入 LLM 并流式返回。

## 自测问题

1. YAML 为什么强调缩进？
2. `-` 在 YAML 中表示什么？
3. 内联提交和档案持久化各有什么优缺点？
4. 为什么“档案在后端”不等于“LLM 自动知道”？
5. `/api/v1/chat/stream` 的核心流程有哪些步骤？

## 答疑和自测问题答案

1. 缩进定义父子关系，决定字段归属。
2. `-` 表示列表项。
3. 内联简单但重复传输；档案持久化更适合长期与多端。
4. LLM 只看到请求上下文，除非把档案摘要显式注入。
5. 读取 prompt → 校验 API Key → 调用 LLM → SSE 输出。

## 下一步建议

- 打开 `rules.yaml`，标出你最想调整的 3 个字段。
- 如果需要个性化输出，尝试“档案摘要注入 system prompt”。
